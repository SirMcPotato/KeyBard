# Contributing to KeyBard

Thank you for your interest in contributing to KeyBard!

## How to Contribute

1. **Fork the repository** and create your branch from `main`.
2. **If you've added code** that should be tested, add tests.
3. **Ensure the project builds** by running `dotnet build`.
4. **Submit a pull request**.

## Style Guidelines

- Follow standard C# coding conventions.
- Use meaningful variable and method names.
- Keep methods focused on a single task.

## Reporting Issues

Use the GitHub issue tracker to report bugs or suggest features. Please provide as much detail as possible, including steps to reproduce bugs.
